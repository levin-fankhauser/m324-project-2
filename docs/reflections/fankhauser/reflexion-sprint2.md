# Inhalt
## 1.1. Welche verschiedenen Themen beinhaltete der Sprint?
In diesem Sprint, welcher von Woche 5 bis Woche 8 dauerte, haben wir die folgenen Themen angeschaut. Als erstes haben wir uns Testing angeschaut. Dort haben wir uns das automatisierte Testen, ursprünglich mit Karma, angaschaut. Für unser Projekt verwenden wir Vitest. Dies ermöglich uns Unit Tests zu den einzelnen Komponenten zu schreiben und so ihre Logik automatisch zu prüfen. Dazu haben wir noch Cypress als E2E Testing Framework eingebaut. Mit Cypress können wir E2E Tests schreiben und so die funktionen unserer App über verschiedene Komponenten hinweg testen. Dies wird beides automatisiert in einem github workflow, damit bei jedem Push sichergestellt wird, dass der gepushte Code fehlerfrei ist. Dazu haben wir jeweils einen workflow für Vitest und Cypress Tests. Wir haben ausserdem Docker angeschaut und ein lauffähiges Docker Image erstellt. Dies haben wir mit docker compose umgesetzt und der Next Standalone funktion, welche einen für docker images optimierten build ausführt um möglichst kleine Abhängigkeiten zu haben. Daz haben wir noch das ganze deployment und das erstellen der Infrastruktur auf AWS angeschaut. Wir haben mit terraform die AWS umgebung vorbereitet um später darauf zu deployen mit kamal. Dies sind 2 Workflows. Einer fürs Terraformen, also die ganze umgebung vorzubereiten. Dieser wird manuel ausgeführt, wenn dies gewünscht ist. Der zweite Workflow ist der Deployment workflow, welcher automatisch durch einen Release von Release-Please gestartet wird und das entsprechenede DOcker Image buildet und dann auf der AWS Instanz startet.

## 1.2. Was haben die Themen mit dem Begriff DevOps zu tun?
Diese Themen sind alle sehr nah mit DevOps verbunden. Automatisiertes Testen ist wichtig um frühzeitig Fehler zu finden und eine korrekt funktionierende Applikation zu gewährleisten. Docker ist wichtig um Images zu builden, welche Ressourcenfreundlich auf einer Produktiven instanz gestartet werden können und das deployment und terraforming ist wichtig um die Applikation produktiv bereitzustellen. Diese sind alles relevante prozesse, da sie einen manuellen prozess ablösen. Alles wird automatisiert und muss nicht mehr von hand gemacht werden, was menschliche Fehler (wie Flüchtigkeitsfehler) vermeidet. Dies hilft bei der Entwicklung und spart bei den meisten ein wenig grösseren Projekten imens an Zeit.
